Deploy · GitHub Pages + WordPress v12 · Jun 2

Self-contained tool that fetches daily-refresh data from a JSON file in the same repo. Path #1 ready today; path #2 (auto-cron) ready to enable when stable.

Files in this deploy package

index.html
The tool. Single file, 203 KB. Fetches live_data.json on load with fallback to inlined defaults if fetch fails.
live_data.json
FIFA rank + Elo + BETSIE values that change. ~5 KB. Updated daily.
update_live_data.py
Daily updater script. Pulls Elo, optionally FIFA, preserves BETSIE.
github_actions_daily_update.yml
Path #2: scheduled auto-update via GitHub Actions. Move to .github/workflows/ to enable.

Step 1 · Push to GitHub Path #1

1
Create repo at github.com/new. Name it wc-2026-tool (or whatever). Public. Initialize with README.
2
Upload 3 files to repo root: index.html, live_data.json, update_live_data.py. Drag-and-drop in browser or use git push. Commit to main.
3
Enable Pages. Settings → Pages → Source: "Deploy from a branch" → Branch: main, folder: / (root) → Save. Wait 1-2 min. URL appears: https://USERNAME.github.io/wc-2026-tool/
4
Verify the tool loads. Open the URL in a new tab. Open browser DevTools Console — should see "Live data loaded · updated Jun 2". If fetch fails (e.g., file 404), tool still works with inlined defaults.
What just happened: GitHub Pages serves your static files at a public HTTPS URL. The tool fetches ./live_data.json from the same origin on each page load. When you update the JSON, every visitor gets the new data within seconds (no caching delay).

Step 2 · Embed in WordPress

1
Add a Custom HTML block to your page or post.
2
Paste this: 
<iframe
  src="https://USERNAME.github.io/wc-2026-tool/"
  width="100%"
  height="2800"
  frameborder="0"
  style="border:0;display:block;width:100%;max-width:1200px;margin:0 auto;"
  allowfullscreen>
</iframe>
Replace USERNAME. Publish.
iframe height: 2800 fits the Matchup tab fully expanded on desktop. If your WordPress theme is narrow, try 2200. Form tab (with squad collapsibles open) needs more — users can tap to collapse if iframe is short.

Step 3 · Daily refresh routine Path #1 active

~2 minutes daily during tournament. Or 5 min weekly pre-tournament since FIFA ranks update monthly.
1
Pull the repo locally: 
git clone https://github.com/USERNAME/wc-2026-tool.git
cd wc-2026-tool
pip3 install requests beautifulsoup4   # one-time
2
Run the update: 
python3 update_live_data.py             # Elo only
python3 update_live_data.py --fifa      # Elo + FIFA
python3 update_live_data.py --dry-run   # preview, no changes
3
For BETSIE updates (manual): when Action Network reruns BETSIE, open live_data.json in any text editor, paste new percentages into the betsie_win and betsie_path blocks. Save.
4
For FIFA refresh (manual override): create a fifa_ranks.json file with the latest ranks: 
{
  "Spain": 1, "Argentina": 2, "France": 3,
  "England": 4, "Colombia": 5, "Brazil": 6,
  ...
}
Then run python3 update_live_data.py --fifa.
5
Push: 
git add live_data.json
git commit -m "daily update $(date +%Y-%m-%d)"
git push
Tool updates within 30 seconds on next page load. No WordPress changes needed.

Step 4 · Enable Path #2 (optional, hands-off mode) Path #2 ready

GitHub Actions auto-runs the update script daily at 7am ET. You stop doing the daily routine entirely — just monitor that it's working.
1
In your local repo, create the folder: 
mkdir -p .github/workflows
mv github_actions_daily_update.yml .github/workflows/daily-update.yml
2
Edit the file: uncomment the schedule: block (remove the # from 2 lines). Save.
3
Push: 
git add .github/workflows/daily-update.yml
git commit -m "enable daily auto-update"
git push
4
Verify: repo → Actions tab → "Daily Live Data Update" should appear in the workflow list. Click "Run workflow" to trigger manually as a smoke test. Subsequent runs happen automatically at 11:00 UTC daily.
When path #2 fires: workflow runs Python script → fetches Elo → commits updated live_data.json → push back to main. Tool sees new data within seconds. Cron timing: 11:00 UTC = 7:00 AM ET = 4:00 AM PT.

Troubleshooting

IssueFix
Tool loads but no "Live data loaded" loglive_data.json wasn't fetched. Check it's in repo root, GitHub Pages served it (open URL .../live_data.json directly), and there's no CORS issue (same-origin should always work)
Tool shows old FIFA ranks after updateBrowser cache. Hard refresh (Ctrl+Shift+R / Cmd+Shift+R). Tool already sends cache: no-cache on fetch, so subsequent visits get fresh data
Squad clubs missingSquad enrichment is a separate workstream — see Squads tab in database. Tool currently shows player names + positions only. Clubs/age/caps coming in v11 of database
Friendly results not showing in toolData is inlined in the tool itself, not in live_data.json. To refresh: re-extract from spreadsheet, rebuild tool, redeploy. Will be a daily-ish task during friendly window (Jun 2-10)
GitHub Actions workflow failsMost likely: eloratings.net scraping broke (their JS-rendered table is fragile). Workflow has continue-on-error: true so it won't block. Update script can be improved with a more stable Elo source if needed
WordPress iframe shows blankSome WordPress security plugins block iframes from external domains. Whitelist *.github.io in plugin settings, or self-host the tool instead
